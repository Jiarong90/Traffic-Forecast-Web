# 摄像头实现指南

## 当前摄像头功能位置

后端：
- `camera1/server.js`
- `camera1/py/compute_engine.py`

前端：
- `UI 2/script.js`

## 当前功能

系统当前支持：
- Map View 中展示实时摄像头点位
- Dashboard 中展示事故相关摄像头证据
- Route Planner 确认导航后展示路线相关摄像头点位
- Alert Detail 中展示相关摄像头信息

## 数据来源

当前主要使用：
- data.gov.sg Traffic Images API
- 其他公开点位数据用于辅助匹配

## 事故与摄像头匹配

当前事故与摄像头匹配逻辑已经迁到 Python 计算层。

基本逻辑：
1. 获取实时事故点
2. 获取实时摄像头点
3. 计算事故附近摄像头
4. 优先保留合理距离内的摄像头
5. 将匹配结果返回给前端展示

当前匹配限制里包含距离阈值，避免把太远的摄像头误匹配到事故。

## 地图展示

Map View 中：
- 摄像头点位可点击查看实时图片
- 事故点位可点击查看详情

Route Planner 中：
- 确认路线后，可在地图上看到与该路线相关的摄像头和事故点位
- 摄像头 popup 与 Map View 保持一致
